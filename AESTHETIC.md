# Aesthetic Style: Webcore / Internetcore

## Overview

This website uses a Webcore-inspired aesthetic to modify the original class website template. Webcore is an internet aesthetic connected to early web culture, nostalgic user interfaces, browser windows, pop-ups, pixel graphics, bright colors, and surreal digital spaces.

The goal of this redesign is not to create a polished corporate website, but to make the page feel like a small personal internet artifact. I wanted it to feel playful, slightly strange, and connected to the visual language of older websites and early operating systems.

## Visual References

The design is inspired by:

- Early personal websites
- GeoCities-era layouts
- Windows XP / old browser interfaces
- Pop-up windows
- Pixel-like borders
- Bright primary colors
- Low-resolution digital graphics
- Internet nostalgia

## Design Choices

### Color

The website uses bright and slightly clashing colors such as:

- Electric blue
- Hot pink
- Pale yellow
- Soft cyan
- White

These colors create a nostalgic internet feeling while still keeping the page readable.

### Typography

The website uses a monospace font to create a computer-terminal and early-web feeling. This makes the page feel less like a modern portfolio website and more like a small archived web page.

### Composition

The main content is placed inside a browser-like window. The top bar, address bar, and small colored buttons reference older desktop interfaces.

### Interaction

The JavaScript button creates a message when clicked. The interaction is intentionally simple because the goal is to demonstrate basic JavaScript while matching the Webcore theme.

### Components

This website includes:

- Browser-style header
- Address bar
- Main content window
- Scrolling marquee text
- Interactive JavaScript button
- Message display box
- Fake pop-up window

## AI Assisted Coding Process

I used AI-assisted coding to help modify the original website template through an iterative "vibe coding" process. I defined the aesthetic direction as Webcore / Internetcore and asked for changes related to color, composition, typography, borders, interaction, and nostalgic browser interface elements.

The original structure of the HTML, CSS, and JavaScript template was kept simple so that I could still understand how each file works.
